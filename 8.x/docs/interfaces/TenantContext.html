    Interface TenantContext

    The resolved, validated program tenant for a single request (component C5, DBX-04 §9.1).

    Tenant identity is resolved and validated before authorization (closes T-01/T-54) and is then carried immutably into the operation. Every field is readonly and the object is deep-frozen at construction (freezeTenantContext), so no downstream layer (authorizer C4, store C6, audit) can mutate a resolved tenant — the immutability the architecture requires (DBX-04 §6, decision 1).

    interface TenantContext {
        audience?: string;
        boxId: string;
        boxRoot: string;
        organisation: string;
        origin?: string;
        program: string;
        relationshipId: string;
        serviceIdentity?: string;
        tenantId: string;
    }
