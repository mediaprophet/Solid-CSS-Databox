    Interface TenantBindingRegistry

    The authoritative registry of per-program tenant bindings (component C5 configuration, DBX-11).

    It answers, for each program-bound fact, "which single tenant owns it?". The cardinal invariant is enforced structurally at registration: no origin, audience or service identity may be bound to more than one tenant, so no platform-wide data-plane credential can exist (T-31) — an attempt to register a cross-tenant fact fails closed. Lookups never reveal anything for an unknown value (enumeration fails safely).
