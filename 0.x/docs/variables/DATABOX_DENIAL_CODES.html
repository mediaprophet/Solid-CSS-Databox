    Structured, audit-safe denial reason codes for the composed Databox authorizer (component C4, DBX-04 §2; DBX-14). Each code names which conjunct of the authorization conjunction (tenant ∧ token-audience ∧ relationship ∧ credential ∧ assurance ∧ delegation ∧ ODRL ∧ immutability) forced the request to deny.

    A reason code is written to the C13 evidence deny event (ADR-0013 §Privacy: "the audit reason codes must themselves avoid leaking protected facts"). It therefore NEVER carries resource content, resource existence, a customer identifier, or any protected fact — only the abstract conjunct that failed. The safe HTTP surface (see ./SafeStepUpResponse) is derived separately and also hides existence (404-not-403).

    • ReadonlyodrlProhibited: "databox:odrl-prohibited"

    • ReadonlyodrlUnsupported: "databox:odrl-unsupported"

    • ReadonlyrelationshipInactive: "databox:relationship-inactive"

      The connection relationship is not active (DBX-13 per-request status).

    • ReadonlytenantMismatch: "databox:tenant-mismatch"

    • ReadonlytokenAudienceMismatch: "databox:token-audience-mismatch"

      The presented token's audience is not bound to the resolved tenant (DBX-11 hard conjunct).
